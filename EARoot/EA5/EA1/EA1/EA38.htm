Below is a brief overview of common top-level directories in the Linux file system and their purposes. Note that exact layouts can vary by distribution, but most adhere to the Filesystem Hierarchy Standard (FHS) or something close to it.<br/><br/>Root Directory (/)<br/>The top-level directory of the entire Linux system. Everything resides under /.<br/>Key Directories<br/>/bin<br/><br/>Stands for “binaries.”<br/>Contains essential user-level commands/binaries (e.g., ls, cp, mkdir) needed for the system to operate in single-user mode.<br/>/sbin<br/><br/>Contains “system binaries” — commands typically reserved for the system administrator (e.g., fsck, shutdown).<br/>/etc<br/><br/>Stands for “et cetera,” but conventionally understood as the system-wide configuration directory.<br/>Nearly all system configuration files (for services like ssh, cron, etc.) and daemon settings live here, e.g. /etc/ssh/sshd_config, /etc/fstab.<br/>/var<br/><br/>Stands for “variable.”<br/>Contains variable data that changes frequently (log files, caches, lock files, spool files like mail, printer spool, etc.).<br/>Example: /var/log contains system and service log files.<br/>/usr<br/><br/>Stands for “Unix System Resources.”<br/>Houses the majority of userland software and libraries (e.g., /usr/bin, /usr/lib). Historically separated to store read-only user programs.<br/>/usr/local is often where locally compiled or manually installed software goes (when using default install prefixes).<br/>/lib and /lib64<br/><br/>Contains essential shared libraries needed by programs in /bin and /sbin.<br/>In modern systems, you’ll also see /usr/lib for additional libraries, with library names like libc.so.<br/>/home<br/><br/>Contains the home directories of regular users (e.g., /home/alice, /home/bob).<br/>Each user’s personal files, documents, and configurations go here.<br/>/root<br/><br/>The home directory for the root (administrator) user.<br/>Separate from /home for security and structural reasons.<br/>/tmp<br/><br/>Temporary files that do not need to persist across reboots.<br/>Often cleared on reboot. Non-critical short-lived data is stored here.<br/>/dev<br/><br/>Device files representing hardware and virtual devices (e.g., /dev/sda for your hard disk, /dev/tty for terminals).<br/>/boot<br/><br/>Contains the kernel and boot loader files (e.g., GRUB files, vmlinuz).<br/>/proc<br/><br/>A virtual filesystem providing process and kernel information (e.g., /proc/cpuinfo, /proc/meminfo).<br/>/sys<br/><br/>Another virtual filesystem providing system and kernel device information.<br/>/opt<br/><br/>For “optional” or third-party software, sometimes used by proprietary or large, self-contained packages.<br/>/media and /mnt<br/><br/>Directories for mounting removable media (/media) and temporary mount points (/mnt).<br/>Example: when you plug in a USB drive, it might auto-mount to /media/username/USB_DRIVE.<br/>---------------------------------------------------------------------------------<br/><br/>When are these directories used?<br/>/etc: Whenever system or service-wide configuration is required. For example, sshd_config (SSH server config) or nginx.conf (webserver config) go here.<br/>/var: Whenever the system or services need to store data that changes often. Logs go in /var/log; spool directories like mail queues go in /var/spool.<br/>/config (if it exists in your setup): Used if the system or a particular application is designed to read settings from a non-standard location. (Again, not typical on standard distributions.)<br/><br/>
