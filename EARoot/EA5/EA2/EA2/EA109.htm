cd host<br/>mkdir build<br/>cd build<br/>cmake ../<br/>make -j4<br/>sudo make install<br/>sudo ldconfig<br/><br/>---------------------------------------------------------------------------------------<br/><br/>1. cmake ../<br/>What it does:<br/>CMake is a build system generator. When you run cmake ../, you’re telling CMake to look in the parent directory (../) for a file called CMakeLists.txt, which describes how the software should be built.<br/><br/>CMake checks for required libraries, checks your compiler settings, and sets up all the project’s configuration and build files (e.g., Makefiles) accordingly.<br/>Why ../?<br/>This is typically run from a dedicated build directory. By providing ../, you are instructing CMake to look in the folder above (host in your case) for the main CMakeLists.txt file. This keeps your build artifacts separate from the source files.<br/><br/>2. make -j4<br/>What it does:<br/>make reads the Makefiles created by CMake and compiles the software. The -j4 argument tells make to use four parallel jobs (threads) during compilation, leveraging up to four CPU cores. If your system has more (or fewer) cores, you can adjust this number.<br/><br/>Why parallel jobs?<br/>Parallelizing the build process speeds up compilation, especially on multi-core machines. For example, using -j8 on an 8-core system can greatly reduce compile time.<br/><br/>3. sudo make install<br/>What it does:<br/>This command installs the compiled binaries, headers, and libraries to the appropriate system directories (often /usr/local/bin, /usr/local/lib, etc.).<br/><br/>sudo is needed because system-wide installation typically writes to directories that require administrator privileges.<br/>Aftermath:<br/>Once installed, your system knows where to find the new software, so you can run commands (like uhd_find_devices) from the terminal.<br/><br/>4. sudo ldconfig<br/>What it does:<br/>ldconfig updates the system’s “library cache,” which tells the OS where all the shared libraries are located. Whenever you install a new library in system directories, you should run ldconfig so the system can properly load them at runtime.<br/>--------------------------------------------------------------------------------------------<br/>By default, sudo make install places files in the system’s “install prefix,” which is most often /usr/local unless otherwise changed. In a typical CMake-based project without custom settings, you’ll end up with:<br/><br/>Executables in \usr/local/bin<br/>Libraries in \usr/local/lib<br/>Header files in \usr/local/include<br/>Shared data in \usr/local/share<br/>
